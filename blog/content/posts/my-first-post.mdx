どっかでブログ始めたいと思ったけどサービス使うのもなぁと思い、静的サイトジェネレータなるものを調べたところ

- 自分で機能拡張しやすい
- マークダウンで書ける
- バージョン管理ができる
- やめるときもマークダウンなので移行もしやすそう
- サーバ契約しなくて良い
- gatsbyの情報が豊富
- gatsby-thema-blogが楽そう

ということで良さげでしたのでgatsbyでブログを作成しました。

以下を参考にしました。すぐ試せました○

[Gatsby で gatsby-theme-blog を使うときの tips](https://gotohayato.com/content/502/)

僕の場合はgithub pagesのリポジトリでソースコードを管理しつつそのまま公開しているので
ブログのディレクトリを指定しないとうまく動作しないみたいです。
gatsbyでビルドするとpublicに出力されるのでそこを指定します。

```js:title=gatsby-config.js
module.exports = {
  pathPrefix: '/blog/public', // highlight-line
  plugins: [
    {
      resolve: `gatsby-theme-blog`,
      options: {},
    },
  ]
};
```

ビルドするときは下のコマンド

```sh:
gatsby build --prefix-paths
```
